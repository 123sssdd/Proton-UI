# Button 按钮

按钮组件用于触发操作和事件，提供多种风格的视觉效果和完整的交互反馈。

## 无障碍访问

Button 组件遵循 WAI-ARIA 规范，提供完整的无障碍访问支持。

### 键盘导航

<Callout type="tip" title="键盘操作">
  - **Tab**: 聚焦到按钮 - **Enter** 或 **Space**: 触发按钮点击事件 - **Shift +
  Tab**: 移动到上一个可聚焦元素
</Callout>

### ARIA 属性

Button 组件会自动添加适当的 ARIA 属性：

- `aria-busy`: 当 `loading={true}` 时设置为 `true`，告知屏幕阅读器按钮正在处理
- `aria-disabled`: 当 `disabled={true}` 或 `loading={true}` 时设置为 `true`
- `role="status"`: 加载图标包含此属性，用于播报加载状态

### 焦点指示器

所有按钮都有清晰的焦点指示器（focus ring），确保键盘用户能够清楚地看到当前焦点位置。

### 颜色对比度

所有按钮变体都经过测试，确保文本与背景的颜色对比度符合 WCAG 2.1 Level AA 标准（至少 4.5:1）。

### 触摸目标

按钮的最小高度为 40px（中号）或 44px（移动端小号），确保在触摸设备上易于点击。

<Callout type="success" title="无障碍访问最佳实践">
  - 始终为按钮提供有意义的文本内容 - 避免仅使用图标的按钮，如果必须使用，请添加
  `aria-label` 属性 - 确保按钮的点击区域足够大（至少 44x44px） -
  在表单中使用适当的 `type` 属性（`submit`、`reset` 或 `button`）
</Callout>

## 设计令牌

Button 组件使用以下 CSS 变量，可以通过主题系统进行自定义：

- `--color-accent`: 主要按钮的背景色
- `--color-bg-primary`: 主要按钮的文本色
- `--color-error`: 危险按钮的背景色
- `--color-text-primary`: 文本颜色
- `--color-text-secondary`: 次要文本颜色

## 最佳实践

### 按钮文本

- 使用清晰、简洁的动词描述操作（如"保存"、"删除"、"提交"）
- 避免使用模糊的文本（如"确定"、"取消"）
- 保持文本简短，通常不超过 3-4 个字

### 加载状态

在执行异步操作时使用 `loading` 状态，防止重复提交：

<DemoBox
  title="异步操作示例"
  description="点击按钮后显示加载状态"
  showCode={true}
  code={`import { Button } from "@proton-ui/core";
import { useState } from "react";

function AsyncButton() {
const [loading, setLoading] = useState(false);

const handleClick = async () => {
setLoading(true);
// 模拟异步操作
await new Promise(resolve => setTimeout(resolve, 2000));
setLoading(false);
};

return (

<Button loading={loading} onClick={handleClick}>
提交表单
</Button>
);
}`}
language="tsx"

>

  <Button loading>提交表单</Button>
</DemoBox>

### 危险操作确认

对于危险操作（如删除），建议使用二次确认：

<Callout type="warning" title="危险操作提示">
  在执行不可逆的操作前，应该： 1. 使用 `danger` 变体突出显示风险 2.
  提供二次确认对话框 3. 在确认对话框中清楚说明操作的后果
</Callout>
